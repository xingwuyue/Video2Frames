# Sprite Sheet 半自动精修工具设计规格

日期：2026-06-04

## 目标

构建一个 Windows 本地浏览器工具，用于把 AI / 渲染生成的视频处理成 Godot 4.x 可用的 sprite sheet。工具重点解决抽帧、抠背景、统一锚点、最大包围盒、手动精修、背景检查和 Godot 导出。

第一版目标不是一键批量转换，而是“半自动精修”：自动完成重复处理，保留人工检查和局部修正能力，优先保证游戏动画资产干净、稳定、可导入。

## 已确认决策

- 工具形态：半自动精修工具。
- 运行方式：本地 Web 工具，浏览器界面操作，本地后端处理重任务。
- 快捷启动：Windows 桌面快捷图标，双击启动本地服务并自动打开浏览器。
- 视频来源：AI / 渲染生成视频，背景色可控。
- 背景策略：默认绿底；角色含绿色时改用红紫或蓝底；支持白底和自定义取样。
- 处理对象：角色、怪物、站立单位、技能特效、飞行物都要支持。
- 锚点策略：支持脚底中心、画面中心、顶部中心、自定义点。
- 基础画布：默认 256x256；动作超出时不裁切，按最大包围盒自动扩展。
- 目标引擎：Godot 4.x。

## 非目标

第一版不做以下内容：

- 云端上传处理。
- 完整桌面软件打包。
- AI 自动生成新动作或补帧。
- 复杂真实拍摄背景的一键抠图。
- 完整 Godot 编辑器插件。
- 多人协作和云项目。
- 跨平台安装包。

## 方案选择

选择“本地 Web 工具 + FFmpeg/OpenCV 后端”。

浏览器界面负责交互：视频导入、参数设置、时间轴预览、锚点调整、洋葱皮检查、橡皮擦、背景切换和导出操作。

本地后端负责重处理：项目文件管理、FFmpeg 抽帧、OpenCV/图像算法抠图、透明边界检测、最大包围盒计算、sprite sheet 拼接和导出文件生成。

该方案比纯前端工具更稳定，尤其适合大视频、精确抽帧和图像批处理；比完整桌面应用范围更小，第一版更容易落地。

## 架构

### 组件

- 浏览器前端
  - 项目入口、视频导入、参数面板。
  - 帧时间轴、单帧预览、sprite sheet 预览。
  - 锚点调整、洋葱皮、画布缩放和平移。
  - 普通橡皮擦、智能区域擦除、恢复笔刷。
  - 背景检查和导出面板。

- 本地后端服务
  - 提供本地 HTTP API。
  - 管理项目目录和缓存。
  - 调用 FFmpeg 抽帧。
  - 调用 OpenCV 或等价图像处理库做抠图、边缘处理、包围盒计算。
  - 生成 PNG、JSON 和 Godot 4.x 辅助文件。

- 快捷启动器
  - Windows 桌面快捷方式。
  - 启动本地服务。
  - 自动打开浏览器到工具地址。
  - 后续可增加停止服务和打开输出目录入口。

## 功能模块

### 1. 项目与视频导入

- 新建项目。
- 打开已有项目。
- 选择源视频。
- 读取视频帧率、尺寸、时长和总帧数。
- 设置抽帧规则：每 N 帧截取一帧。
- 保存原始视频路径和抽帧配置。

### 2. 背景抠除

- 默认使用绿底抠图。
- 支持红紫、蓝、白、自定义取样背景。
- 提供容差参数。
- 提供基础边缘清理和溢色处理。
- 导入后评估背景纯度：
  - 适合抠图。
  - 可能有边缘污染。
  - 建议更换背景色或提高生成视频质量。

### 3. 帧管理时间轴

- 展示抽取后的帧。
- 显示原始视频帧号。
- 支持选择、隐藏、删除、重排。
- 支持重新按新间隔抽帧。
- 保留原始抽帧缓存，不破坏源数据。

### 4. 包围盒与画布

- 对每一帧透明区域计算 alpha 包围盒。
- 对当前动作序列计算最大包围盒。
- 默认单元格尺寸为 256x256。
- 如果最大包围盒无法放入 256x256，则自动扩展单元格尺寸。
- 扩展时不裁掉角色或特效。
- 导出前提示最终单元格尺寸。

### 5. 锚点与洋葱皮

- 支持锚点预设：
  - 脚底中心：适合角色、怪物、站立单位。
  - 画面中心：适合技能特效和飞行物。
  - 顶部中心：适合特殊挂点类资源。
  - 自定义点：适合非标准对象。
- 支持整体锚点调整。
- 支持单帧锚点微调。
- 支持洋葱皮叠帧检查动画漂移。
- 锚点配置写入项目文件和导出 JSON。

### 6. 精修编辑器

精修阶段必须支持视图控制和两类擦除方式。

视图控制：

- 放大和缩小。
- 鼠标滚轮缩放。
- 缩放百分比显示。
- 放大后平移画布。
- 适配窗口。
- 100% 原始比例显示。
- 高倍率下可显示像素网格。

擦除工具：

- 普通橡皮擦
  - 圆形笔刷。
  - 可调大小。
  - 可调硬度。
  - 适合精确擦边。

- 智能区域擦除
  - 用户点击残留色块。
  - 按点击点周围的相近颜色和连通区域匹配。
  - 默认只擦除与点击点连通的相近颜色区域，不全图删除同色。
  - 支持容差调节。
  - 支持擦除前高亮预览。
  - 支持应用和取消。
  - 支持撤销。

- 恢复笔刷
  - 从原始抠图帧恢复误擦区域。
  - 不直接修改原始抽帧。

精修记录保存为 mask 或编辑操作记录。单帧精修和 sprite sheet 级精修都需要可追踪；如果在生成后的 sheet 上擦除，导出时需要映射回对应帧区域。

### 7. 背景检查

- 透明棋盘背景。
- 白色背景。
- 绿色背景。
- 红紫背景。
- 背景切换只影响预览，不写入最终透明 PNG。

### 8. 图集布局

- 用户设置 X 行 X 列。
- 按当前帧顺序拼接。
- 不足格保持透明。
- 如果帧数超过容量，导出前阻止并提示：
  - 增加行数。
  - 增加列数。
  - 删除不需要的帧。
  - 后续版本可支持自动分页。

### 9. Godot 4.x 导出

导出内容：

- `sheet.png`：透明 sprite sheet。
- `frames.json`：帧元数据。
- 可选 `frames/`：单帧透明 PNG。
- 可选 Godot 4.x 导入辅助脚本或配置。

`frames.json` 至少包含：

- sheet 文件名。
- 单元格宽高。
- 行数和列数。
- 帧总数。
- 每帧在 sheet 中的 `x`、`y`、`w`、`h`。
- 每帧原始视频帧号。
- 每帧锚点。
- 当前动作 FPS 或建议播放帧率。
- 最大包围盒。
- 背景抠除参数摘要。

Godot 4.x 目标用法：

- 可通过 `AnimatedSprite2D` / `SpriteFrames` 使用导出的帧。
- 也可通过 `Sprite2D + AnimationPlayer` 使用帧区域。
- 像素风资源应提示使用最近邻过滤，避免边缘发糊。

## 数据流

1. 源视频导入
   - 保存源视频路径。
   - 读取帧率、尺寸、时长、总帧数。

2. 抽帧
   - 使用 FFmpeg 按每 N 帧抽取。
   - 生成 `frames/raw/`。
   - 每张图保留原始视频帧号。

3. 抠图
   - 根据背景颜色、容差和边缘参数生成 alpha。
   - 生成 `frames/keyed/`。
   - 保留抠图参数，允许重新计算。

4. 精修
   - 用户对单帧或 sheet 进行擦除、恢复。
   - 保存编辑记录或 mask。
   - 原始帧和初始抠图帧不被破坏。

5. 包围盒
   - 对当前有效帧计算透明边界。
   - 计算最大包围盒。
   - 决定最终单元格尺寸。

6. 锚点归一
   - 根据锚点预设和微调值定位每一帧。
   - 保证播放时轴心稳定。

7. 图集拼接
   - 按 X 行 X 列生成 sheet。
   - 应用精修结果。
   - 不足格透明。

8. 导出
   - 生成 `sheet.png`。
   - 生成 `frames.json`。
   - 按配置生成单帧 PNG 和 Godot 辅助文件。

## 项目目录

建议项目结构：

```text
project-name/
  project.json
  source/
    source-video.mp4
  frames/
    raw/
    keyed/
  edits/
  exports/
    sheet.png
    frames.json
```

`project.json` 保存：

- 源视频路径。
- 抽帧间隔。
- 背景色和容差。
- 边缘处理参数。
- 有效帧列表和顺序。
- 锚点预设和逐帧微调。
- 画布尺寸。
- 行列布局。
- 导出配置。

## 错误处理与保护规则

### 背景不纯

如果背景颜色波动过大，工具提示：

- 重新生成更干净的纯色背景视频。
- 改用手动背景取样。
- 调整容差。
- 换成红紫或蓝底。

### 角色含背景色

如果角色或特效明显包含当前背景色，工具提示抠图风险，并建议更换背景色或降低容差。

### 动作超出 256x256

工具不裁切角色。它应自动扩展单元格尺寸，并在导出前明确显示最终尺寸。

### 帧数超过布局容量

导出前阻止导出，并提示用户增加行列、删除帧或后续使用分页。

### 误擦除

必须支持撤销和恢复笔刷。擦除记录不直接破坏原始帧。

### 导出失败

保留项目文件和缓存。显示失败步骤和日志，允许重新导出。

### 硬性保护

- 不覆盖源视频。
- 不破坏原始抽帧。
- 不因尺寸不足强行裁切角色。
- 导出前检查透明边界、帧数量、行列容量和锚点配置。

## 验收标准

第一版必须通过以下场景：

- Windows 桌面快捷图标能启动工具并打开浏览器。
- 能导入 AI / 渲染生成的绿底视频。
- 能按每 N 帧抽取帧，并保留原始视频帧号。
- 能使用绿底抠图生成透明帧，无明显大块绿色残留。
- 当角色含绿色时，可以改用红紫或蓝底策略。
- 当动作超出 256x256 时，自动扩展单元格，不裁掉角色。
- 人物/怪物可用脚底中心锚点导出，播放时无明显漂移。
- 特效/飞行物可用画面中心锚点导出，播放时视觉中心稳定。
- 精修编辑器支持放大、缩小、平移、适配窗口和 100% 显示。
- 普通橡皮擦可用。
- 智能区域擦除支持容差、预览、应用、取消和撤销。
- 恢复笔刷可恢复误擦区域。
- 背景检查可切换透明棋盘、白色、绿色、红紫色。
- X 行 X 列布局能正确拼接 sheet。
- 帧数超过布局容量时能阻止导出并提示。
- 导出的 `sheet.png` 有正确透明通道。
- 导出的 `frames.json` 包含帧矩形、锚点、帧序和原始视频帧号。
- 项目关闭后重新打开，参数和精修记录不丢失。

## 后续增强

- 动作预设：idle、walk、attack、hurt、death。

- 自动分页导出多个 sheet。
- Godot 4.x 编辑器插件。
- 更高级的边缘半透明处理。
